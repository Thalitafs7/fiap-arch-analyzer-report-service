A

AnalysisAggregate

Bounded Context: Diagram Analysis

Raiz de Agregado

  • ROOT
    AnalysisAggregate Controla ciclo de vida da analise. Campos: id, status, file_name, file_type, s3_key, sqs_message_id, diagram_file, extraction_result, error_message

Entidades

  • ENTITY
    ExtractionResult Resultado da extracao via LLM Vision. Contem: components[], relationships[], patterns[], raw_description

Value Objects

  • VO
    DiagramFile Arquivo validado e codificado em base64. Max 20MB. Campos: file_name, file_type, media_type, content_base64, file_size_kb
  • VO
    Component Componente arquitetural identificado (frozen, imutavel)
  • VO
    Relationship Relacao entre componentes: source → target: description
  • VO
    ArchitecturalPattern Padrao identificado (ex: microservices, event-driven)

Enumeracoes

  • ENUM
    AnalysisStatus RECEIVED → PROCESSING → ANALYZED | ERROR
  • ENUM
    FileType PNG | JPEG | GIF | WEBP | PDF

Operacoes

  • CMD
    create()Factory: cria no estado RECEIVED
  • CMD
    start_ingestion(diagram_file)RECEIVED → PROCESSING
  • CMD
    complete_extraction(result)Registra ExtractionResult
  • CMD
    complete(qa_score)PROCESSING → ANALYZED
  • CMD
    fail(step, error_message)Qualquer → ERROR
R

ReportAggregate

Bounded Context: Report Generation

Raiz de Agregado

  • ROOT
    ReportAggregate Controla geracao e validacao de qualidade. Campos: id, analysis_id, report, qa_score

Entidades

  • ENTITY
    TechnicalReport Relatorio gerado. Campos: components_identified[], architectural_risks[], recommendations[], executive_summary, rag_used

Value Objects

  • VO
    RiskItem Risco com: category, description, severity, affected_components[], mitigation
  • VO
    Recommendation Recomendacao arquitetural, com flag rag_influenced
  • VO
    QAScore Score de qualidade: is_valid, completeness_score (≥0.6), issues_found[], quality_notes
  • VO
    RagContext Contexto RAG: has_context, enrichment_text, similar_analyses_count

Enumeracoes

  • ENUM
    RiskCategory SPOF | Seguranca | Escalabilidade | Acoplamento | Observabilidade | Resiliencia
  • ENUM
    Severity ALTO | MEDIO | BAIXO

Domain Services

  • SVC
    GuardrailService Valida grounding (anti-alucinacao ≤20%), completude de componentes, recomendacoes nao-vazias, sumario ≥100 chars

Operacoes

  • CMD
    create(report_id, analysis_id)Factory: cria ReportAggregate vazio
  • CMD
    attach_report(technical_report)Vincula TechnicalReport e emite ReportGeneratedEvent
  • CMD
    attach_qa(qa_score)Vincula QAScore e emite QAValidationCompletedEvent

Shared Kernel

ID AnalysisId UUID - identifica uma analise
ID ReportId UUID - identifica um relatorio
BASE DomainEvent Classe base com occurred_at (UTC) e event_name
DiagramReceivedEvent
Analysis BC — emitido por AnalysisAggregate.create()
Payload: analysis_id, file_name, source (upload|sqs)
DiagramIngestedEvent
Analysis BC — emitido por start_ingestion()
Payload: analysis_id, file_type, file_size_kb
ComponentsExtractedEvent
Analysis BC — emitido por complete_extraction()
Payload: analysis_id, components_count, patterns_count
ReportGeneratedEvent
Report BC — emitido por attach_report()
Payload: report_id, analysis_id, rag_used, risks_count
QAValidationCompletedEvent
Report BC — emitido por attach_qa()
Payload: report_id, analysis_id, is_valid, completeness_score
AnalysisCompletedEvent
Analysis BC — emitido por complete()
Payload: analysis_id, qa_score
AnalysisFailedEvent
Analysis BC — emitido por fail()
Payload: analysis_id, step, error_message (pode ocorrer em qualquer ponto)

ia-service :8000

Servico principal de analise IA (FastAPI). Consome SQS, executa pipeline de extracao, risco e geracao de relatorio.
  • PostgreSQL + pgvector
  • OpenAI API (GPT-4o Vision)
  • AWS SQS / S3
  • HuggingFace (fine-tuning opcional)
  • LangChain (JsonOutputParser)

report-api :8001

API de consulta de relatorios gerados. Leitura do banco PostgreSQL.
  • PostgreSQL (leitura)

streamlit-app :8501

Interface web para upload de diagramas e visualizacao de resultados.
  • ia-service (HTTP)
  • report-api (HTTP)

PostgreSQL + pgvector :5432

Banco relacional com extensao vetorial. Armazena analyses, extraction_results, reports e embeddings.
  • pgvector/pgvector:pg16
  • Volume: pgvector_data
  • Healthcheck: pg_isready
  • Indice HNSW para busca vetorial

AWS SQS

Fila de mensagens para integracao assincrona com sistema SOAT externo.
  • Long polling
  • Max 5 mensagens/batch
  • Idempotencia por message_id

AWS S3

Armazenamento de diagramas referenciados nas mensagens SQS.
  • Download com retry 3x
  • Suporta: PNG, JPEG, GIF, WEBP, PDF
  • Limite: 20MB por arquivo
